Measuring LLM Temperament via Contrastive Hidden-State Probing

We calibrate linear direction vectors from contrastive pairs of hidden states (warm vs cold, formal vs casual, etc.), then project neutral responses onto these axes to extract a stable personality fingerprint per model. We validate with test-retest ICC, measure steerability by injecting axis vectors at inference, and test robustness under adversarial multi-turn conflict.

Fingerprints

Each model's default personality measured on 50 neutral questions without any style instruction. Values range from −1 (left pole) to +1 (right pole). Click cards to select for comparison.

Method: For each of 5 bipolar axes (warm/cold, confident/cautious, etc.), we calibrate a direction vector by contrasting hidden states under opposing style instructions (50 positive + 50 negative prompts). The axis vector is the normalized mean difference between the two groups, extracted from the last 4 transformer layers. We then ask 50 neutral questions with no style instruction, extract hidden states, and project them onto each axis. The mean projection is the model's baseline personality score on that axis.

Measurement Stability

Do we get the same fingerprint if we measure again with different questions? Higher cosine similarity = more reproducible. ≥0.8 excellent, 0.6–0.8 moderate, <0.6 poor.

Method: We run calibration 3 times independently (sets A, B, C) using different random subsets of 30 questions each. Each set produces its own axis vectors. We compute pairwise cosine similarity between axis vectors from different sets (A↔B, A↔C, B↔C) and average. A cosine of 1.0 means identical directions; 0.0 means unrelated. Models with high d-prime (strong separation between contrastive states) produce more stable axes.
Why do some models score low? Stability depends on d-prime — the separation between contrastive hidden states. Models with low d-prime (e.g. GPT-OSS 20B, d'≈3) produce noisy axis vectors that fluctuate across independent calibration sets. GPT-OSS is a MoE model with only 3.6B active parameters and a small hidden dimension (2880) — different prompts activate different expert subsets, adding noise. In contrast, models like Qwen3 8B (d'≈12) produce highly stable axes regardless of which questions are used. Nemotron-H and Granite 8B also score low for similar reasons — low d-prime, not architecture per se.

Steering: Optimal Layer

At which transformer layer does steering have the strongest effect? We test every layer and pick the one that moves the personality projection the most.

Method: For each model, we test 10–15 evenly spaced layers (25% to 95% depth). At each layer, we inject the axis direction vector scaled by α×h_norm (where h_norm = median hidden state norm at that layer) and generate responses to 5 questions at α = −5, 0, +5. The “effect” is the projection difference between α=+5 and α=−5. Key finding: optimal layers are typically at 25–65% depth. Late layers (>90%) often show near-zero effect because alignment training has “cemented” the output.
Key finding: alpha normalization unlocks steering. Hidden state norms vary by 5,000× across models (from ~10 for Qwen 0.5B to ~51,000 for Gemma 3 12B). Without normalization, alpha=5 is a 0.01% perturbation for Gemma — invisible. With h_norm scaling (direction × median hidden state norm), alpha=1 means the same relative perturbation for every model. Result: all models now show steering effect >0.4, up from only 5 models previously. Optimal layers cluster at 25–65% depth for most models.

Steering Playground

Drag the slider to inject a personality vector into the model and see how the response text changes in real time.

Method: We add α×(v×h_norm) to the hidden states at the optimal layer during autoregressive generation, where v is the unit axis vector, h_norm is the median hidden state norm (makes α comparable across models), and α controls the strength. Two-pass evaluation: Pass 1 generates text with the steering hook active. Pass 2 re-encodes the steered text through the clean model to measure where it falls on all 5 axes. This separates "what the model wrote" from "how we judge it." Each question shows 7 alpha values (−5, −2, −1, 0, +1, +2, +5) with the actual generated text.

Conflict Drift

Does the model's personality change when users push back aggressively? Lower drift = more robust identity under pressure.

Method: We simulate 50 adversarial multi-turn conversations (12–16 turns each). The user progressively escalates conflict — sarcasm, dismissal, direct contradiction of the model's style. After each turn, we extract hidden states and project onto the 5 personality axes. “Drift” is the mean absolute change in projection from turn 1 to turn 12. Plus 10 neutral control conversations to separate drift caused by conflict from natural conversation dynamics.
